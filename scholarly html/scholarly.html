1. Abstract

We want to develop a web application based on microservices that can perform searching and filtering of images from a certain dataset using trained models to identify and visualize different characteristics. The main framework that interacts with microservices we want to use is Flask because of its integration with Python language and simplicity over other full-stack frameworks like Django. For the client side we intend on using ReactJS library for the easy integration with Flask. The backend part will be written in Python and should include machine learning algorithms on data visualization and identification implemented using the Keras API, data storing and retrieval using the H5py interface for easy manipulation using Keras, and SPARQLWrapper as an endpoint interface for Python.

The 'Best Artworks of All Time' is structured as follows:

  • images folder containing full sized paintings, sequentially numbered, structured in a separate folder for each artist
  • resized folder containing the same paintings, sequentially numbered, but in a smaller size (useful for machine learning) and extracted from their respective folders into a single folder
  • artists.csv file containing basic information about the 50 chosen artists

The data in artists.csv has the following columns:

  • name: name of the artist
  • years: birth year and the death year, separated by a dash
  • genre: the genres of the paintings created by the artist
  • nationality: ethnic group of the artist
  • bio: short description of the artist (taken from the artist's wikipedia page, usually a paragraph or two)
  • wikipedia: link to the artist's wikipedia page
  • paintings: number of paintings gathered in the artist's respective folder

NOTE: the dataset is slightly modified from the original one because of character encoding issues... The artists.csv file had a lot of modifications done to it (the bio column being basically redone from scratch). As for the paintings, 'Albrecht Durer' had two sets of identical files, but with slightly different names (due to loss of original encoding), so we kept only one set, renamed to the correct artist name.

4.1 Flask

Flask is a web framework written in Python, based on micro-services. Flask makes the core functionality simple yet extensible, saving time to create web applications. It does not require any separate tools or libraries, but it is based on two popular engines: Werkzeug (collection of WSGI libraries) and Jinja (template engine).

Flask works with two main folders:

  • template, which contains HTML files to be parsed by Flask as needed (can contain Jinja templates).
  • static, which contains miscellaneous files that do not change such as CSS stylesheets, JS scripts and resources (images, text etc.).

Features of Flask:

  1. based on the Unicode encoding
  2. uses REST API
  3. Development server (seeing real-time changes without redeploying)
  4. Support for unit testing
  5. Client side secure cookies
  6. Mature documentation
  7. High compatibility with other frameworks
  8. Lots of extensions available which enrich the functionalities

Great thing about Flask is that it is a portable and interactive web development language. It is easy to get into it and understand the few powerful functions that are the 'bread and butter' of the framework, while also giving the ability to scale into complex applications. It offer freedom of choice, not enforcing the use of any dependency. The developer chooses what they want to use in terms of external tools and libraries.

ReactJS

ReactJS is a component-based library which is deployed for the development of interactive user interfaces. Currently it is the most popular front-end JS library. It incorporates the view (V) layer in the MVC (Model View Controller) pattern. It is supported by Facebook, Instagram and a community of individual developers and organizations. React basically enables development of large and complex web-based applications which can change their data without subsequent page refreshes. It targets to provide better user experiences and with blazing fast and robust web apps development.

Features of React:

  1. Lightweight DOM for Better Performance - it does not interact with the DOM generated by the browser but reacts to the document object model stored in the memory which results in a blazing fast and robust performance of the application
  2. Easy Learning Curve - a developer gets along with the framework very easily
  3. JSX - highly popular language which is very similar to the tech giant XML, not mandatory but helps in making development easier, whenever writing mark-ups for components and the corresponding binding events
  4. Performance – highly efficient framework because it uses a virtual DOM feature
  5. One-way data flow – it is designed in such a way that unidirectional data flow that is downstream is allowed and supported, making it well-known for the generation of canonical data sources that remains synchronized across the components that which pay attention to it
  6. Virtual DOM - key feature, similar to the document object model generated by the browser but with a difference that it is stored it memory. Whenever a request for changing the page content is made, the changes are reflected to the memory residing virtual DOM first. After that, a diff() algorithm compares the virtual DOM and the browser DOM an then the required changes only are reflected to the browser DOM, instead of re-rendering the entire DOM. This provides a gigantic boost to the performance of application, essentially when thousands of data changes are to be made

The Component is the major building-block in React. The entire user interface can be designed by assembling a tree of multiple components. An intermediate DOM is generated by the render() method present in each of the React components. As shown in Fig.1, a call to the React.renderComponent() method on the root Component leads to recursively going down the Component tree and generating the intermediate-DOM. Then afterwards this intermediate DOM is converted to the actual HTML DOM.

React makes use of a convenient XML-based extension to JavaScript, known as the JSX, to generate the component tree as a combination of multiple XML nodes. This makes DOM visualization and understanding easy and much more convenient. JSX also plays a key role in simplifying the association of the event handlers and properties as XML attributes. The final JavaScript is generated using a command line and in browser tool. A Component is directly mapped by a JSX XML node. It is important to note that React works independent of JSX and the using JSX only plays the task of simplifying the task of generating intermediate DOM.

React has a few limitations too that must be considered:

  • Only the View entity in the mobile view controller or MVC is handled by the react. Therefore, additional tooling is required in order to complete the project development
  • Use of inline templates and JSX sometimes proves to be an intensively complex and tiring task for a small set of developers
  • Also, in case of ReactJS, failures happen at compile time instead of runtime as in case of other languages and frameworks, which can sometimes be very frustrating and tiring

Keras API

Keras is a deep learning API written in Python, running on top of the machine learning platform TensorFlow. It was developed with a focus on enabling fast experimentation, being able to go from idea to result as fast as possible is key to doing good research. It offers consistent & simple APIs, it minimizes the number of user actions required for common use cases, and it provides clear and actionable feedback upon user error.

Keras ranked as #1 for deep learning both among primary frameworks and among all frameworks used.

The core data structures of Keras are layers and models. The simplest type of model is the Sequential model, a linear stack of layers. For more complex architectures, the Keras functional API allows to build arbitrary graphs of layers, or write models entirely from scratch via subclasssing.

H5py interface

The h5py package is a Pythonic interface to the HDF5 binary data format.

It lets you store huge amounts of numerical data, and easily manipulate that data from NumPy. For example, you can slice into multi-terabyte datasets stored on disk, as if they were real NumPy arrays. Thousands of datasets can be stored in a single file, categorized and tagged however you want.

H5py uses straightforward NumPy and Python metaphors, like dictionary and NumPy array syntax. For example, you can iterate over datasets in a file, or check out the shape or dtype attributes of datasets. You don't need to know anything special about HDF5 to get started.

In addition to the easy-to-use high-level interface, h5py rests on an object-oriented Cython wrapping of the HDF5 C API. Almost anything you can do from C in HDF5, you can do from h5py.

Best of all, the files you create are in a widely-used standard binary format, which you can exchange with other people, including those who use programs like IDL and MATLAB.
